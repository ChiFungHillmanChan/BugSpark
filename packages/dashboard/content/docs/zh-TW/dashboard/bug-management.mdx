# 錯誤管理

錯誤頁面提供兩種檢視模式來管理報告：**表格** 和 **看板**。

## 表格檢視

預設的表格檢視列出所有錯誤報告，具有可排序的欄位：

- **ID** — 唯一的追蹤 ID（例如 `BSK-42`）
- **標題** — 錯誤摘要
- **嚴重程度** — `critical`、`high`、`medium` 或 `low`
- **狀態** — 目前的工作流程狀態
- **建立時間** — 相對時間戳

點擊任何列可開啟完整的錯誤詳情頁面。

## 看板檢視

切換到看板模式以獲得視覺化的工作流程面板。報告按狀態組織成欄：

- **新建** — 剛提交，尚未審查
- **分類中** — 正在調查
- **處理中** — 正在積極修復
- **已解決** — 修復已部署
- **已關閉** — 已驗證並歸檔

## 篩選

使用篩選列來縮小結果範圍：

- **搜尋** — 在標題和描述中進行文字搜尋
- **狀態標籤** — 切換一個或多個狀態篩選
- **嚴重程度標籤** — 切換一個或多個嚴重程度篩選
- **日期範圍** — 最近 7 天、30 天、90 天或所有時間
- **清除** — 重置所有篩選

## 錯誤詳情頁面

點擊報告會開啟詳情頁面，包含：

- **中繼資料面板** — 嚴重程度、狀態、負責人、回報者、時間戳、裝置資訊
- **螢幕截圖檢視器** — 在原始和標註螢幕截圖之間切換
- **主控台日誌** — 可展開的已擷取主控台條目列表
- **網路瀑布流** — HTTP 請求的時間軸和計時資料
- **工作階段時間軸** — 使用者操作的時間順序列表
- **留言討論串** — 團隊討論和備註

## 更新狀態

使用中繼資料面板中的下拉選單更改報告狀態。可用的轉換：

| 從 | 到 |
|------|------|
| new | triaging、in_progress、resolved、closed |
| triaging | in_progress、resolved、closed |
| in_progress | resolved、closed |
| resolved | closed、in_progress（重新開啟） |
| closed | in_progress（重新開啟） |

## 指派報告

從負責人下拉選單中選擇團隊成員來指派報告。被指派者會收到通知，報告會出現在其篩選檢視中。
